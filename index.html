01 — Concepto Clave

¿Qué es el mantenimiento
de un sistema informático?

El mantenimiento de sistemas informáticos abarca todas las actividades técnicas, administrativas y de gestión durante el ciclo de vida de un sistema, con el fin de mantenerlo o restaurarlo a un estado en que pueda cumplir la función requerida.

💻

Sistema Informático

Conjunto integrado de hardware, software, firmware y datos que procesa, almacena y comunica información. Incluye servidores, redes, dispositivos de usuario y toda la infraestructura digital asociada.

HW + SW + FW + DATA = Sistema
🔧

Mantenimiento

Proceso continuo para prevenir fallos, corregir errores y optimizar el rendimiento. Va desde actualizaciones de firmware hasta limpieza física de componentes y revisión de logs de eventos.

Prevenir → Detectar → Corregir
🛡️

Aseguramiento de Operación

Conjunto de controles técnicos y de proceso que garantizan la disponibilidad (A), integridad (I) y confidencialidad (C) de los sistemas bajo cualquier condición operativa.

Tríada CIA: A + I + C
📊

SLA / Niveles de Servicio

Acuerdo formal que define tiempos de respuesta, disponibilidad mínima y métricas de calidad. Parámetros como RTO (Recovery Time Objective) y RPO (Recovery Point Objective) son centrales.

RTO < 4h · RPO < 1h
02 — Clasificación

Tipos de Mantenimiento

01
🟢

Preventivo

Se ejecuta antes de que ocurra un fallo. Incluye limpieza de hardware, actualización de drivers, análisis de disco y chequeos periódicos de temperatura.

  • Limpieza física de componentes
  • Actualización de firmware / drivers
  • Análisis S.M.A.R.T. de discos
  • Revisión de logs del sistema
Proactivo
02
🟠

Correctivo

Actúa después de detectar un fallo. Puede ser inmediato (emergencia) o diferido (planificado). Involucra diagnóstico, reparación o reemplazo de componentes.

  • Diagnóstico de fallo
  • Reparación o sustitución
  • Pruebas de regresión
  • Documentación del incidente
Reactivo
03
🔵

Predictivo

Usa monitoreo en tiempo real y análisis de datos para anticipar fallos antes de que sucedan. Emplea telemetría, análisis de tendencias y machine learning.

  • Monitoreo continuo de métricas
  • Análisis de tendencias (trending)
  • Alertas tempranas automatizadas
  • Modelos de predicción de fallos
Inteligente
04
🟣

Evolutivo / Adaptativo

Actualiza el sistema para adaptarse a nuevos requisitos del entorno (SO, cambios normativos, nuevas amenazas). Incluye migraciones y mejoras de seguridad.

  • Migración de plataformas
  • Actualizaciones de seguridad
  • Adaptación normativa
  • Escalado de capacidad
Evolutivo
03 — Arsenal Técnico

Herramientas Esenciales

Hardware💾

CrystalDiskInfo

Monitorea la salud de HDDs y SSDs vía S.M.A.R.T.. Lee temperatura, horas de uso, errores realocados y predice fallos inminentes.

Estado: ✓ Bueno | Temp: 34°C
Red📡

Wireshark

Analizador de protocolos de red. Captura y disecciona paquetes en tiempo real para detectar anomalías, cuellos de botella y tráfico malicioso.

$ wireshark -i eth0 -k
Sistema📋

Event Viewer / Journalctl

Registro centralizado de eventos del SO. Permite auditar errores, advertencias y eventos de seguridad para diagnóstico y cumplimiento normativo.

$ journalctl -p err -b
Rendimiento📈

Nagios / Zabbix

Plataformas de monitoreo continuo de infraestructura. Miden CPU, RAM, disco, red y servicios. Disparan alertas cuando los umbrales se superan.

CRITICAL: CPU > 95% (5m)
Seguridad🔐

ClamAV / Defender

Motores antimalware para escaneo, detección y cuarentena de amenazas. Claves para el mantenimiento preventivo de seguridad en estaciones y servidores.

$ clamscan -r /home --bell
Backup☁️

Veeam / rsync

Soluciones de respaldo y recuperación. Garantizan el RPO y RTO definidos en el SLA mediante copias incrementales, diferenciales y completas.

$ rsync -avz /src user@bak:/dst
04 — Metodología

Proceso de Mantenimiento

1

Inventario y Diagnóstico

Levantamiento completo de activos: hardware, software y licencias. Se identifican versiones, estados y dependencias. Base para cualquier plan de mantenimiento efectivo.

Evaluación inicial
2

Planificación y Calendarización

Definición de ventanas de mantenimiento, responsables, procedimientos y criterios de éxito. Se minimizan impactos al negocio mediante mantenimientos en horario de baja demanda.

Planificación
3

Respaldo Previo (Backup)

Antes de cualquier intervención se realizan copias de seguridad verificadas. Si algo falla durante el proceso, se puede restaurar al estado previo sin pérdida de datos.

Seguridad de datos
4

Ejecución Controlada

Aplicación de los cambios siguiendo checklist definidos. Actualizaciones, parches, limpieza, configuraciones. Cada acción queda registrada en el log de mantenimiento.

Implementación
5

Pruebas y Verificación

Post-mantenimiento: pruebas funcionales, de rendimiento y de seguridad. Se verifica que todos los servicios respondan correctamente y que los KPIs estén en parámetros esperados.

Control de calidad
6

Documentación y Mejora Continua

Registro de todo lo realizado, incidencias encontradas y lecciones aprendidas. La documentación alimenta el ciclo PDCA para mejorar futuros mantenimientos.

PDCA · Mejora
05 — Recomendaciones

Buenas Prácticas

01

Regla 3-2-1 de Backups

Mantén 3 copias de tus datos, en 2 tipos de medios diferentes, con 1 copia offsite. Es la estrategia más robusta para garantizar la recuperación ante cualquier escenario.

02

Principio de Mínimo Privilegio

Asigna solo los permisos estrictamente necesarios para cada usuario y servicio. Limita la superficie de ataque y reduce el impacto de un posible compromiso de credenciales.

03

Parches y Actualizaciones

Mantén un ciclo de parcheo definido. Las vulnerabilidades críticas deben parchearse en menos de 72h. Prueba siempre en entorno de staging antes de aplicar en producción.

04

Monitoreo y Alertas Proactivas

Configura umbrales de alerta tempranos (CPU >80%, disco >85%, RAM >90%). Recibir alertas antes del colapso te da margen para actuar sin impacto al usuario final.

05

Documentación Actualizada

Mantén siempre actualizado el inventario de activos, diagramas de red y procedimientos. En caso de incidente crítico, la documentación precisa reduce el MTTR drásticamente.

06

Plan de Recuperación ante Desastres

El DRP (Disaster Recovery Plan) define los pasos exactos para restablecer operaciones tras un fallo catastrófico. Debe probarse al menos una vez al año con simulacros documentados.
